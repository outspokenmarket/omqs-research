Matrix multiplication in classification training

Now the problem is binary classification. The structure is still the same: the input matrix X is multiplied by the weight matrix W. That produces a score, also called a logit: z = XW. Then we pass that score through the sigmoid function to get probabilities: p = σ(z).

So the matrix multiplication still drives the forward pass. The only difference is that, instead of predicting a continuous number directly, the model predicts the probability that each example belongs to class 1. Then we compare those probabilities with the target labels, compute the loss, compute the gradient, and update the weights.

why matrix multiplication still matters in classification

The points below live in feature space. The orange arrow is the weight vector W. The dashed line is the decision boundary, where z = x · W = 0. On one side, the model leans toward class 1. On the other side, it leans toward class 0.

The key idea is unchanged: one matrix multiplication processes the whole batch at once. It computes all logits simultaneously, then the sigmoid converts them into probabilities. Training adjusts the same weight vector so the probabilities move closer to the correct labels.
